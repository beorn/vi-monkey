# AI Mode

AI mode uses LLMs to intelligently explore your system, finding edge cases that random fuzzing might miss.

## Quick Start

```bash
vitest ai                        # AI exploration
vitest ai board.spec.ts          # specific file
vitest ai --save                 # save findings as tests
vitest ai --model=gpt-4          # specify model
```

## How It Works

Unlike fuzz mode's random action selection, AI mode uses an LLM to:

1. **Observe** the current state
2. **Reason** about interesting actions to try
3. **Execute** the chosen action
4. **Reflect** on the result
5. **Repeat** with accumulated knowledge

```
┌─────────────────────────────────────────────────────────────┐
│                       AI Test Loop                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   state ──► LLM ──► "try pressing 'h' at boundary"          │
│              │                                              │
│              ▼                                              │
│          action: 'h'                                        │
│              │                                              │
│              ▼                                              │
│   Surface.sendAction(instance, 'h')                         │
│              │                                              │
│              ▼                                              │
│   new state ──► check invariants                            │
│              │                                              │
│         ┌────┴────┐                                         │
│         │         │                                         │
│       pass      fail                                        │
│         │         │                                         │
│         ▼         ▼                                         │
│      continue   report + save                               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## Configuration

### Vitest Config

```typescript
// vitest.config.ts
import { vitestx } from 'vitestx/plugin'

export default defineConfig({
  plugins: [
    vitestx({
      ai: {
        // Model selection (Vercel AI SDK format)
        model: 'claude-sonnet',

        // Temperature (0 = deterministic, good for CI)
        temperature: 0,

        // Maximum actions per exploration
        maxSteps: 50,

        // Token budget (prevent runaway costs)
        maxTokens: 10000,

        // Save discovered tests
        saveDir: './tests/generated',
      },
    })
  ]
})
```

### Environment Variables

```bash
# API keys (model-dependent)
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...

# Override model
AI_MODEL=gpt-4

# Override temperature
AI_TEMPERATURE=0

# Disable AI mode in CI (cost control)
AI_ENABLED=false
```

## Spec Requirements

AI mode uses the same spec format as fuzz mode:

```typescript
// board.spec.ts
import { defineSurface, defineInvariants } from 'vitestx'

export const surface = defineSurface({
  create: () => renderBoard(SAMPLE_DATA),
  sendAction: (board, key) => board.press(key),
  getState: (board) => board.getState(),
  query: (board, sel) => board.query(sel),
})

export const actions = ['j', 'k', 'h', 'l', 'Enter', 'Escape']

export const invariants = defineInvariants({
  singleCursor: (s) => s.queryAll('[data-cursor]').length === 1,
  cursorInBounds: (s) => s.cursor >= 0 && s.cursor < s.items.length,
})

// Optional: describe the system for better AI reasoning
export const description = `
  A keyboard-navigable board with items.
  - j/k: move cursor down/up
  - h/l: move cursor left/right
  - Enter: select item
  - Escape: deselect
`
```

## AI Exploration Strategies

The LLM uses several strategies:

### Boundary Testing
```
"The cursor is at index 0. Let me try 'k' to see if it goes negative."
```

### State Machine Exploration
```
"I've seen states A and B. Let me try to reach state C by pressing Enter."
```

### Invariant Probing
```
"The invariant says there should always be one cursor. Let me try rapid
 Escape/Enter sequences to see if I can create a state with no cursor."
```

### Error Message Analysis
```
"The last action caused a warning about 'invalid parent'. Let me explore
 what combinations of actions trigger this."
```

## Saving Discovered Tests

When AI mode finds interesting behavior (not just failures), it can save them:

```bash
vitest ai --save
```

Generated test:

```typescript
// tests/generated/board-cursor-boundary.test.ts
// Generated by vitestx AI mode
// Found: 2024-01-15T10:30:00Z
// Model: claude-sonnet

import { test, expect } from 'vitest'
import { surface } from '../board.spec'

test('cursor stays in bounds at top boundary', () => {
  const instance = surface.create()

  // Move to top
  surface.sendAction(instance, 'k')
  surface.sendAction(instance, 'k')
  surface.sendAction(instance, 'k')

  // Try to go above
  surface.sendAction(instance, 'k')

  const state = surface.getState(instance)
  expect(state.cursor).toBeGreaterThanOrEqual(0)
})
```

## CI Integration

### Deterministic Mode

For CI, use temperature=0 and save outputs:

```yaml
# .github/workflows/test.yml
- name: AI exploration
  run: vitest ai --save
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
    AI_TEMPERATURE: 0
    AI_MAX_TOKENS: 5000

- name: Commit generated tests
  run: |
    git add tests/generated/
    git commit -m "chore: add AI-discovered tests" || true
```

### Cost Control

Token budgets prevent runaway costs:

```typescript
vitestx({
  ai: {
    maxTokens: 5000,      // per file
    maxTotalTokens: 50000, // per run
  },
})
```

### Replay Mode

Replay saved AI sessions deterministically:

```bash
vitest ai --replay=.ai-sessions/board-2024-01-15.json
```

## Claude Code Integration

If you're using Claude Code, you can leverage your existing subscription:

```typescript
vitestx({
  ai: {
    provider: 'claude-code',  // Uses Claude Code's API
  },
})
```

This avoids separate API costs and uses your Claude Code context.

## Comparison: AI vs Fuzz

| Aspect | Fuzz Mode | AI Mode |
|--------|-----------|---------|
| Action selection | Random (weighted) | Intelligent (LLM) |
| Coverage | Broad, statistical | Targeted, semantic |
| Speed | Fast (1000s/sec) | Slow (10s/sec) |
| Cost | Free | API costs |
| Reproducibility | Seed-based | Temperature=0 |
| Finds | Statistical edge cases | Semantic edge cases |

**Use fuzz mode for**: High-volume testing, CI, regression

**Use AI mode for**: Exploratory testing, finding semantic bugs, generating test ideas

## Advanced: Custom Prompts

Customize the AI's exploration strategy:

```typescript
vitestx({
  ai: {
    systemPrompt: `
      You are testing a financial application.
      Focus on:
      - Boundary conditions (zero, negative, overflow)
      - Race conditions (rapid actions)
      - State consistency (balances must match)
    `,

    actionPrompt: (state, actions, history) => `
      Current state: ${JSON.stringify(state)}
      Available actions: ${actions.join(', ')}
      Previous ${history.length} actions: ${history.map(h => h.action).join(' → ')}

      What action should we try next and why?
    `,
  },
})
```

## Limitations

- **Cost**: LLM API calls cost money
- **Speed**: Much slower than random fuzzing
- **Determinism**: Even with temperature=0, results may vary slightly
- **Context window**: Long sessions may exceed token limits

AI mode is best used for targeted exploration, not exhaustive testing.
